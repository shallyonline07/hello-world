=== ng-contianer ====
        - The ng-container does not render in the DOM. Only the content inside it is rendered. It is a syntex element.
        - It makes the DOM light weight
        - We usually use it to render inline content.
        - We cann't apply css classes.
Without ng-container:
{{ data. name }}
Without ng-container: It removed an extra div from DOM
{{ data. name }}

=== ng-template ====
        - The ng-template defines a template that is not rendered by default.
        - You can re-use the content inside ng-template multiple times by using *ngTemplateOutlet directive
        - You can also provide data to ng-template using context
        - We cann't apply css classes.
ng-template without context: ng-template with context:
I'm a div
I'm a label

I'm a paragraph

Welcome {{ name }}. You age is {{ age }}
